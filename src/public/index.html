Capabilities

Built for Production

Six core pillars that make this engine internship-assessment ready and real-world scalable.

Multi-Tenant Isolation

Each user receives a unique UUID session token. Cart operations are strictly scoped to the authenticated tenant — zero cross-talk guaranteed.

x-session-token

Tiered Promo Engine

Dynamic campaign rules with Bronze (5%), Silver (10%), Gold (15%) tiers plus a 2% diversity bonus for shopping across 3+ categories.

promotionService.js

Rate Limiting

100 requests per 15-minute window per IP via express-rate-limit. Registration endpoints throttled to 10 req/15min to prevent abuse.

express-rate-limit

TTL Cart Expiry

MongoDB TTL index auto-deletes inactive carts after 7 days. Each cart interaction refreshes the expiry — eliminating database bloat.

MongoDB TTL Index

Input Validation

Every request validated via express-validator at the route level. Structured 400 errors with field-level detail returned on any schema mismatch.

express-validator

Structured Logging

Winston + Morgan HTTP logger pipeline writes JSON logs (combined.log, error.log) with timestamps, IPs, method, route, and tenant context.

winston + morgan

Promotion Formula

Subtotal Σ (price × quantity)
Tier % match(subtotal, campaign.tiers)
Diversity % categories ≥ 3 ? 2% : 0%
Final subtotal × (1 − tier% − div%)

MongoDB Schema

Cart {
  userId    → ref(User)
  status    → active | checked_out
  items[]   → embedded subdocs
  expiresAt → TTL(7 days)
}

CartItem {
  productId · name · price
  category · quantity
}
